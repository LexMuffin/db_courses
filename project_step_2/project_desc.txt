Данная структура базы данных представляет собой организацию магазина по продаже компьютерного оборудования(но подобная схема может использоваться не только для этого).
Начну с описания таблиц базы данных.
Таблица products будет содержать товар и его описание, где в таблице catalogs будет содержаться категория товара, в таблице producer наименование производителя товара. В таблице storehouse_procuts будет храниться наличие товара на складе определенного количества, которое есть в наличии.
В таблице order_line будет указываться id заказа покупателя и id товара.
В таблице order будет указываться id покупателя, id продавца и итоговая сумма к оплате.
Также отдельно вынесены таблицы с кратким описанием продавца и таблица должностей в магазине
Определена таблица для клиентов с адресом их проживания, городом и улицей.
Также будет учтено состояние заказа, то есть в таблице order_step будет отслеживаться актуальный этап доставки и состояние заказа покупателя.
